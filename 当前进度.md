现在项目已经有了 **RAG 系统的前半段：知识库构建能力**。

------

每次启动项目，需要按这个顺序：

```bash
cd ~/Desktop/local-rag-system
conda activate RagSystem
```

启动 Qdrant：

```bash
docker compose up -d
```

确认 LM Studio 打开，并且两个模型都 loaded：

```text
qwen/qwen3-v1-8b
text-embedding-bge-small-zh-v1.5
```

启动 FastAPI：

```bash
uvicorn app.main:app --reload
```

测试健康检查：

```bash
curl http://127.0.0.1:8000/health
```

测试上传文档：

```bash
curl -X POST "http://127.0.0.1:8000/documents/upload" \
  -F "file=@项目设计.md"
```

------

你现在的启动流程可以记成：

```text
1. 开 LM Studio
2. 开 Qdrant
3. 开 FastAPI
4. 上传文档
```

对应命令：

```bash
conda activate RagSystem
docker compose up -d
uvicorn app.main:app --reload
```

------

下一步该做的是 **RAG 查询链路**：

```text
用户提问
↓
把问题转成 embedding
↓
去 Qdrant 检索相关 chunks
↓
把 chunks 塞进 prompt
↓
调用本地 Qwen
↓
返回 answer + sources
```

也就是开始实现：

```text
POST /rag/query
```

下一轮我们要写这些文件：

```text
app/schemas/rag.py          # 定义 QueryRequest / QueryResponse
app/rag/retriever.py        # 从 Qdrant 检索相关 chunks
app/llm/local_llm.py        # 调用 LM Studio 的 Qwen
app/rag/prompt.py           # RAG prompt 模板
app/rag/chain.py            # 组装 RAG 流程
app/services/rag_service.py # 业务服务层
app/api/rag.py              # 暴露 /rag/query API
app/main.py                 # 注册 rag router
```

也就是说：**你已经把资料存进知识库了，下一步是让系统能基于知识库回答问题。**
