# 项目说明与进度 — Local RAG System

> 本地 RAG 知识问答系统。上传文档 → 自动向量化入库 → 提问 → LLM 结合知识库回答，全程本地运行。

---

## 启动命令

按顺序执行：

```bash
# 1. 激活 conda 环境
# 2. 启动 Qdrant 向量数据库（Docker）
# 3. 启动 FastAPI 后端（:8000）
conda activate localrag
docker compose up -d
uvicorn app.main:app --reload

# 4. 启动前端（:5173，新终端窗口）
cd frontend
npm run dev
```

> 确保 LM Studio 已打开，且以下两个模型已加载：
> - `qwen/qwen3-v1-8b`（LLM）
> - `text-embedding-bge-small-zh-v1.5`（Embedding）

快速验证：

```bash
curl http://127.0.0.1:8000/health          # 健康检查
curl -X POST "http://127.0.0.1:8000/documents/upload" -F "file=@somefile.pdf"  # 上传文档
```

## 项目现状

### 已完成（4 个 Sprint）

**Sprint 1 — 知识库构建**
- 企业化项目结构搭建
- LM Studio 本地模型接通（LLM + Embedding）
- Qdrant 向量数据库接通
- 文档上传 → 解析 → 切块 → 向量化 → 入库，完整链路

**Sprint 2 — RAG 问答闭环**
- `/rag/query` 问答接口
- 向量检索（Qdrant similarity_search_with_score，支持 top_k 调节）
- RAG Prompt 设计（限制基于 context 回答，防幻觉）
- LangChain LCEL 链：prompt → LLM → output parser
- 答案 + sources 引用（含文件名、页码、相似度分数）
- 全流程 6 步耗时日志

**Sprint 3 — 前端交互层**
- React 19 + Vite 前端工程
- 聊天 UI（类 ChatGPT 交互）
- 文档上传面板（支持 PDF/TXT/MD，含批量上传）
- 文档管理器（按类型分组浏览、删除文档）
- 系统状态指示（健康检查、Top-K 滑块）
- 前后端 CORS 打通
- `GET /documents` — 列出所有已索引文档
- `DELETE /documents/{source}` — 删除文档（同时清理 Qdrant + 本地文件）

**Sprint 4 — 智能查询处理 + 前端视觉升级** 🆕

查询处理（Pre-Retrieval Query Processing）：
- **意图识别**：LLM 自动识别用户意图（问答/总结/对比/事实查找 4 类）
- **查询改写**：LLM 将模糊问题改写为更清晰、更适合向量检索的查询语句
- **失败回退**：LLM 调用失败时静默降级为原始查询，不影响主流程
- **同模型复用**：查询处理与答案生成使用同一个 LLM，不引入额外模型
- 新增 `app/rag/query_processor.py` 模块，修改 `app/services/rag_service.py` 集成

前端视觉升级（"Editorial Ink" 深色主题）：
- 全新 CSS 设计系统：CSS 自定义属性（design tokens）、三层背景深度
- Plus Jakarta Sans 字体（Google Fonts, 300–800 weight），替代系统默认字体
- 更克制的玻璃效果：仅顶栏和输入区使用，侧边栏改为不透明表面
- 消息样式优化：助手消息纯排版无气泡，用户消息保留淡紫色气泡
- 欢迎页 SVG 图标替代 emoji，提示卡片交互优化
- CSS 从 ~2000 行精简至 ~800 行

## 系统架构

```
浏览器 (:5173)
    │
    ▼
FastAPI (:8000)
    ├── /health          → 健康检查
    ├── /documents/upload → 文档上传入库
    ├── /documents       → 文档列表
    ├── /documents/{src} → 删除文档
    └── /rag/query      → RAG 问答
    │
    ▼
RAG Pipeline（LangChain）
    上传链路: loader → splitter → embeddings → Qdrant
    问答链路: query_processor → retriever → prompt → LLM → answer + sources
            （查询改写）   （向量检索）        （答案生成）
    │
    ▼
外部依赖
    Qdrant (Docker, :6333)    — 向量存储
    LM Studio (:1234)         — LLM + Embedding
```

### 代码分层

```
api/        ← 路由层，只处理 HTTP 请求/响应
services/   ← 业务层，编排调用
rag/        ← RAG 核心：loader, splitter, embeddings, query_processor, vectorstore, retriever, prompt, chain
llm/        ← LLM 实例创建，支持本地/云端双模式切换
core/       ← 配置（.env → pydantic-settings）
schemas/    ← Pydantic 请求/响应模型
utils/      ← 文件保存、ID 生成等工具函数
```

## API 一览

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/health` | 健康检查 |
| `GET` | `/documents` | 列出已索引文档 |
| `POST` | `/documents/upload` | 上传文档（multipart） |
| `DELETE` | `/documents/{source}` | 删除文档 |
| `POST` | `/rag/query` | RAG 问答 |

`POST /rag/query` 请求/响应示例：

```json
// Request
{ "question": "什么是 RAG?", "top_k": 5 }

// Response
{
  "question": "什么是 RAG?",
  "answer": "RAG 是检索增强生成...",
  "sources": [
    { "content": "...", "source": "intro.pdf", "page": 3, "score": 0.87 }
  ]
}
```

## 技术栈

| 层 | 技术 |
|----|------|
| 后端框架 | FastAPI + Uvicorn |
| RAG 编排 | LangChain（LCEL 管道语法） |
| 向量数据库 | Qdrant（Docker） |
| LLM | qwen/qwen3-v1-8b（via LM Studio, OpenAI 兼容 API） |
| Embedding | text-embedding-bge-small-zh-v1.5 |
| 文档解析 | PyPDF (PDF) + TextLoader (txt/md) |
| 文本切分 | RecursiveCharacterTextSplitter (chunk=800, overlap=120) |
| 前端 | React 19 + Vite + Axios |
| 配置 | pydantic-settings + .env |
