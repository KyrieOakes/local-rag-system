# Dev Log — 多文件上传功能

> **日期**: 2026-05-09
> **标签**: feature, upload, batch, api, frontend, ui

---

## 一、概述

新增批量文件上传功能，支持用户在一次操作中选择并上传多个文档。后端新增 `/upload-batch` 端点，前端上传面板从单文件选择改为多文件列表管理，支持添加/移除文件、逐文件反馈上传结果、部分成功/失败的友好提示。

## 三、API 设计

### `POST /upload-batch`

**请求**: `multipart/form-data`，字段名 `files`，多个文件

**响应 200**:
```json
{
  "results": [
    {"filename": "report.pdf", "file_path": "data/raw/a1b2.pdf", "pages": 10, "chunks": 45, "status": "indexed"},
    {"filename": "bad.exe", "status": "error", "error": "Unsupported file type: .exe"},
    {"filename": "notes.txt", "file_path": "data/raw/c3d4.txt", "pages": 1, "chunks": 8, "status": "indexed"}
  ]
}
```

- 始终返回 200 — 逐文件结果通过 `status` 字段区分（`"indexed"` 或 `"error"`）
- 文件依序处理（非并行），避免 Qdrant 并发写入问题
- 原有 `POST /upload` 单文件端点保持不变

## 四、实现细节

### 4.1 后端批量端点 (`app/api/documents.py`)

- 新增 `from typing import List` 导入
- 新增 `upload_documents_batch(files: List[UploadFile] = File(...))` 
- 循环处理每个文件：`save_upload_file()` → `ingest_document()`
- `ValueError` 和 `Exception` 被捕获为结果字典中的错误条目，不抛 HTTPException
- 复用所有现有函数（`save_upload_file`、`ingest_document`），无重构

### 4.2 前端 API 层 (`frontend/src/api.js`)

- 新增 `uploadDocuments(files)`：对每个文件调用 `formData.append('files', file)`
- 字段名 `files` 与 FastAPI 参数名匹配
- 返回 `res.data`（即 `{ results: [...] }`）

### 4.3 前端状态变更 (`App.jsx`)

- `file (File|null)` → `files (File[])`：支持多文件选择
- 新增 `uploadResults`：存储后端返回的逐文件结果数组
- `closeUploadPanel()` 清空 `files` 和 `uploadResults`
- `handleUpload()` 调用 `uploadDocuments(files)`，根据结果计算 succeeded/failed 计数，生成 all-success / partial / all-fail 三类摘要
- 上传完成后不自动关闭面板，方便用户查看逐文件状态
- 新增 `getFileResult(filename)` 辅助函数，根据文件名查找对应的上传结果

### 4.4 前端上传面板 UI 重写

**文件输入**:
- 添加 `multiple` 属性
- `onChange` 使用 `Array.from(e.target.files)` 获取所有选中文件
- 新选文件 **追加** 到现有列表（不替换）
- 重置 `e.target.value = ""` 以允许重复选择同一文件

**拖放区域** (`.modal-upload-area`):
- 空状态：保持原有上传图标 + 提示文案（"Click to choose files"）
- 有文件状态 (`has-files`)：紧凑显示 "Click to add more files"

**文件列表** (`.modal-file-list`):
- 滚动容器（最大高度 240px，自定义滚动条）
- 每行显示：文件类型图标 + 文件名 + 文件大小 + 操作按钮
- 上传前：红色圆形 X 按钮，点击移除该文件
- 上传后：绿色 ✓ 或红色 ✗ 状态指示器，hover 显示详情（chunks 数或错误信息）
- 整行绿色/红色边框着色表示成功/失败状态

**上传按钮**:
- 无文件时禁用，显示 "Upload & Index"
- 有文件时显示 "Upload & Index 3 Files"
- 上传中显示 spinner + "Uploading..."

**消息区**:
- 新增 `.partial` 类型（琥珀色），用于部分成功/部分失败场景
- 全成功 → 绿色，全失败 → 红色，部分 → 琥珀色

## 六、测试验证

### 手动测试步骤

1. 打开上传面板 → 点击区域 → 验证文件选择器允许多选
2. 选择 3 个文件（混合 .pdf/.txt/.md）→ 验证列表显示所有文件（名称、大小、图标）
3. 点击某文件的 X 按钮 → 验证文件从列表移除，按钮计数更新
4. 再次点击区域 → 选择更多文件 → 验证追加到现有列表
5. 上传 2 个有效文件 → 验证所有行显示绿色 ✓ + "2 files uploaded..." 绿色消息
6. 上传 1 个有效 + 1 个 .exe（改名为 .txt 但内容无效，或直接用 .exe）→ 验证一个绿色 ✓、一个红色 ✗（hover 显示错误），琥珀色摘要
7. 上传 0 个文件 → 验证按钮禁用
8. 关闭面板 → 重新打开 → 验证列表为空
9. 验证 `npm run build` 编译通过
