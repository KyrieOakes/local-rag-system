# 项目说明与进度 — Local RAG System

> 本地 RAG 知识问答系统。上传文档 → 自动向量化入库 → 提问 → LLM 结合知识库回答，全程本地运行。

---

## 启动命令

按顺序执行：

```bash
# 1. 激活 conda 环境
# 2. 启动 Qdrant 向量数据库（Docker）
# 3. 启动 FastAPI 后端（:8000）
conda activate localrag
docker compose up -d
uvicorn app.main:app --reload

# 4. 启动前端（:5173，新终端窗口）
cd frontend
npm run dev
```

> 确保 LM Studio 已打开，且以下两个模型已加载：
> - `qwen/qwen3-v1-8b`（LLM）
> - `text-embedding-bge-small-zh-v1.5`（Embedding）

快速验证：

```bash
curl http://127.0.0.1:8000/health          # 健康检查
curl -X POST "http://127.0.0.1:8000/documents/upload" -F "file=@somefile.pdf"  # 上传文档
```

## 项目现状

### 已完成（5 个 Sprint）

**Sprint 1 — 知识库构建**
- 企业化项目结构搭建
- LM Studio 本地模型接通（LLM + Embedding）
- Qdrant 向量数据库接通
- 文档上传 → 解析 → 切块 → 向量化 → 入库，完整链路

**Sprint 2 — RAG 问答闭环**
- `/rag/query` 问答接口
- 向量检索（Qdrant similarity_search_with_score，top_k 硬编码为 5）
- RAG Prompt 设计（自然流畅段落式回答，基于 context 防幻觉）
- LangChain LCEL 链：prompt → LLM → output parser
- 答案 + sources 引用（含文件名、路径、chunk_index、页码、相似度分数）
- 全流程 6 步耗时日志

**Sprint 3 — 前端交互层**
- React 19 + Vite 前端工程
- 聊天 UI（类 ChatGPT 交互）
- 文档上传面板（支持 PDF/TXT/MD/DOCX，含批量上传）
- 文档管理器（按类型分组浏览、删除文档）
- 系统状态指示（健康检查）
- 前后端 CORS 打通
- `GET /documents` — 列出所有已索引文档
- `DELETE /documents/{source}` — 删除文档（同时清理 Qdrant + 本地文件）

**Sprint 4 — 智能查询处理 + 前端视觉升级**

查询处理（Pre-Retrieval Query Processing）：
- **意图识别**：LLM 自动识别用户意图（问答/总结/对比/事实查找 4 类）
- **查询改写**：LLM 将模糊问题改写为更清晰、更适合向量检索的查询语句
- **失败回退**：LLM 调用失败时静默降级为原始查询
- 新增 `app/rag/query_processor.py` 模块

前端视觉升级（"Editorial Ink" 深色主题）：
- CSS 自定义属性设计系统，Plus Jakarta Sans 字体
- 玻璃效果顶栏和输入区，侧边栏不透明表面
- CSS 从 ~2000 行精简至 ~800 行

**Sprint 5 — 批量摄入 + 增量更新 + 查询日志**

批量摄入（`ingest.py` + `app/rag/ingestion/`）：
- **CLI 脚本**: `python ingest.py --input_dir data/engineering --batch_size 64`
- **增量更新**: SQLite checksum 数据库，MD5 变更检测，文件未改则跳过
- **批量 embedding**: 直接调用 OpenAI-compatible API (input: list[str])，每批 64 条
- **批量写入**: Qdrant `upsert` 按 500 点/批拆分，避开 32MB payload 限制
- **Markdown 标题保留**: MarkdownHeaderTextSplitter 保留 H1/H2/H3 层级
- **统一 pipeline**: 上传 API 与 `ingest.py` 共用同一套 load→split→batch embed→bulk upsert 流程
- 新增支持格式: `.docx` (Docx2txtLoader), `.markdown`

查询日志（`app/rag/query_logger.py`）：
- **终端输出**: 简短一行摘要（时间、intent、top chunk、答案预览）
- **JSONL 持久化**: `logs/history/rag_queries.jsonl` 记录完整查询轨迹（问题、改写后查询、检索结果含 score + file_path + chunk_index、最终答案）
- SourceChunk 扩展: 新增 `file_name`, `file_path`, `chunk_index` 字段

## 系统架构

```
浏览器 (:5173)
    │
    ▼
FastAPI (:8000)
    ├── /health              → 健康检查
    ├── /documents/upload    → 文档上传入库
    ├── /documents/upload-batch → 批量上传
    ├── /documents           → 文档列表
    ├── /documents/{src}     → 删除文档
    └── /rag/query           → RAG 问答 (top_k=5 硬编码)
    │
    ▼
RAG Pipeline（LangChain）
    摄入链路: loader → splitter → batch_embedder → bulk_writer → Qdrant
              (ingest.py CLI 与上传 API 共用)
    问答链路: query_processor → retriever → prompt → LLM → answer + sources
            （查询改写）   （向量检索）        （答案生成）
    日志链路: query_logger → terminal (brief) + logs/history/rag_queries.jsonl (full)
    │
    ▼
外部依赖
    Qdrant (Docker, :6333)    — 向量存储
    LM Studio (:1234)         — LLM + Embedding
```

### 代码分层

```
api/        ← 路由层，只处理 HTTP 请求/响应
services/   ← 业务层，编排调用
rag/        ← RAG 核心：loader, splitter, embeddings, query_processor, query_logger,
              vectorstore, retriever, prompt, chain, ingestion/
llm/        ← LLM 实例创建，支持本地/云端双模式切换
core/       ← 配置（.env → pydantic-settings）
schemas/    ← Pydantic 请求/响应模型
utils/      ← 文件保存、ID 生成等工具函数
ingest.py   ← CLI 批量摄入脚本
```

## API 一览

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/health` | 健康检查 |
| `GET` | `/documents` | 列出已索引文档 |
| `POST` | `/documents/upload` | 上传文档（multipart） |
| `DELETE` | `/documents/{source}` | 删除文档 |
| `POST` | `/rag/query` | RAG 问答 |

`POST /rag/query` 请求/响应示例：

```json
// Request
{ "question": "什么是 RAG?" }

// Response
{
  "question": "什么是 RAG?",
  "answer": "RAG 是检索增强生成...",
  "sources": [
    {
      "content": "...",
      "source": "intro.pdf",
      "file_name": "intro.pdf",
      "file_path": "data/raw/abc123.pdf",
      "chunk_index": 3,
      "page": 3,
      "score": 0.87
    }
  ]
}
```

## 技术栈

| 层 | 技术 |
|----|------|
| 后端框架 | FastAPI + Uvicorn |
| RAG 编排 | LangChain（LCEL 管道语法） |
| 向量数据库 | Qdrant（Docker） |
| LLM | qwen3-8b-mlx（via LM Studio, OpenAI 兼容 API） |
| Embedding | text-embedding-qwen3-embedding-4b |
| 文档解析 | PyPDF (PDF), TextLoader (txt/md/markdown), Docx2txtLoader (docx) |
| 文本切分 | MarkdownHeaderTextSplitter (H1/H2/H3) + RecursiveCharacterTextSplitter (chunk=800, overlap=120) |
| 前端 | React 19 + Vite + Axios + react-markdown |
| 配置 | pydantic-settings + .env |
| 摄入 | ingest.py CLI + app/rag/ingestion/ (checksum/batch embed/bulk upsert) |
| 日志 | query_logger → JSONL (logs/history/) + terminal |
