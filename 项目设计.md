第 1 周：FastAPI + LangChain + Chroma/Qdrant 跑通 PDF 问答
第 2 周：换成 Milvus + PostgreSQL，做文档上传、删除、查询 API
第 3 周：加 Hybrid Search + Rerank + sources citation
第 4 周：加 LangGraph Agentic RAG、评估集、Docker 部署



核心架构：

```text
Backend: FastAPI
RAG Framework: LangChain
Vector DB: Qdrant
Embedding: nomic-embed-text 或 bge-small/bge-m3
LLM: Qwen3 / Qwen2.5 via LM Studio
Document Parsing: pypdf / unstructured
Config: python-dotenv
Deployment: Docker Compose
```



架构：

```text
用户
 ↓
FastAPI
 ↓
LangChain RAG Pipeline
 ↓
Embedding Model → Qdrant
 ↓
Retriever → Prompt → Local LLM
 ↓
返回 answer + sources
```



项目功能可以定为：

```text
POST /documents/upload     上传 PDF / txt / md
POST /rag/query            问答
GET  /documents            查看文档
DELETE /documents/{id}     删除文档
GET  /health               健康检查
```



后续要拔高，再加：

```text
Hybrid Search
Reranker
LangSmith tracing
RAG evaluation
LangGraph Agentic RAG
```

## 分层原则

这个项目遵循比较标准的后端分层方式：

```text
API Layer
↓
Service Layer
↓
RAG Core Layer
↓
Vector DB / LLM / File System
```

对应到代码：

```text
app/api        接口层
app/services   业务层
app/rag        RAG 核心能力层
app/llm        模型调用层
app/core       配置层
app/schemas    数据结构层
```

------

## 为什么这个结构适合面试

这个结构有几个优点：

1. 职责清晰
   API 不直接操作 Qdrant，RAG 逻辑不混在 main.py 里。
2. 易扩展
   以后从 Qdrant 换 Milvus，只需要改 `vectorstore.py`。
3. 易测试
   每个模块都可以单独写测试。
4. 有工程意识
   有配置、有日志、有测试、有 Docker、有文档。
5. 能讲架构
   面试时可以清楚解释从用户请求到最终回答的完整链路。
